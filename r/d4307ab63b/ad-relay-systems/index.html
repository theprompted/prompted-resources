Setup Guide

The Ad Relay Systems

Point at a funnel. Walk away. Come back to a folder of image ads ready to publish.

  • Generates 20 image ads from a single funnel URL — every image congruent with the funnel's hook
  • Writes the matching ad copy — primary text and headline for each image
  • Outputs a Facebook feed preview HTML so you can scrub the batch before pushing to your ad account

01Pick the version that matches your funnel

Two flavors of the same system. The setup process is identical — they only differ in what kind of funnel they're tuned for. Pick one (or both) and download.

Native-style

Ad Relay

Locks the funnel's headline and orbits every image around it. Best for advertorials, listicles, and native-article landing pages where one headline is doing the heavy lifting.

Use when: your funnel URL looks like /pages/..., /blog/..., or /articles/...

Product-driven

Product Ad Relay

Preserves your actual product photo-accurately across compositions. Best for product detail pages where the product itself is the scroll-stop and needs to render exactly as it appears in real life.

Use when: your funnel URL looks like /products/... or /collections/...

Not sure? Look at the URL of the page you'd link the ads to. If the URL contains the word products or collections, you want the Product Ad Relay. Otherwise, start with the Ad Relay.

02What's in the bundle

Whichever bundle you download, you get everything Claude Code needs to run the relay end-to-end. No external dependencies beyond Python and fal-client (which the setup skill installs for you).

  • The relay skill — invoked with a single phrase in Claude Code
  • The setup orchestrator — installs Python + fal-client, prompts you for your FAL_KEY, copies the skill into place
  • The 7-leg relay engine — bash script + JSON template + image generator + agent instructions
  • The swipe file — 307 ad formats across 15 families with full composition data per format
  • .env template + requirements file for clean setup

03Get started in four steps

You need Claude Code already installed (you're a paid Claude subscriber). That's the only assumption. Claude Code installs everything else for you — Python, the fal-client library, and your API key configuration — automatically.

  1. Download the zip using the button above. Move it to your Desktop and double-click to unzip. You'll get a folder named after the bundle you chose.
  2. Open Claude Code in that folder. In Terminal (Mac) or Command Prompt (Windows): cd ~/Desktop/ad-relay-system && claude (Replace ad-relay-system with product-ad-relay-system if you downloaded that one.)
  3. Tell Claude Code to set up the system. Once you see the ❯ prompt, paste one of: set up the ad relay system set up the product ad relay system Claude Code will detect what's missing, install Python and fal-client, prompt you for your FAL_KEY, and copy the skill into place. It tells you what it's doing at each step.
  4. Run your first relay. Once setup completes, paste one of: run the ad relay on [FUNNEL URL] run the product ad relay on [PDP URL] Claude Code handles the whole pipeline. A 20-concept run takes ~15–30 minutes and costs roughly $1–$3 in fal credits.
→ You'll need a fal.ai key

The relay uses fal.ai for image generation. Free signup, pay-as-you-go (roughly $0.04–$0.10 per image). Setup will prompt you for the key — grab one at fal.ai/dashboard/keys before you start.

⚠ Heads up — password prompts

If your machine doesn't have Python yet, Claude Code may ask for your computer password to install Homebrew (Mac). That's normal — type your password into the Terminal window when you see the prompt. You won't see characters as you type. Press Enter when done.

04If something goes wrong

Just describe what you saw to Claude Code. It can read its own scripts, parse error messages, and walk you through fixes interactively — that's the whole point.

The most common stumble is FAL_KEY not set — usually because you opened a new terminal without sourcing the .env from setup. Run set -a; source .env; set +a and retry. If you see a 401 from fal, your key is bad — regenerate at fal.ai/dashboard/keys. A 402 means you're out of fal credits.

If you're using the Product Ad Relay and the generated product looks redesigned or substituted instead of preserved, the reference image wasn't attached to the generation call. Re-run the affected leg — Claude Code will detect the gap and fix it.

If a leg seems to hang for more than a few minutes, there's usually a stale tail -f still attached to a finished log. Kill it (ps aux | grep tail → kill <pid>) and the next leg picks up.
