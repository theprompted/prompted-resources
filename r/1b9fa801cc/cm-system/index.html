01Get started in four steps

You need Claude Code already installed (you're a paid Claude subscriber). That's the only assumption. Claude Code installs everything else for you — Python, the libraries, the cron schedule — automatically.

  1. Download the zip using the button above. Move it to your Desktop and double-click to unzip. You'll get a folder called contribution-margin-system.
  2. Open Claude Code in that folder. In Terminal (Mac) or Command Prompt (Windows): cd ~/Desktop/contribution-margin-system && claude Windows: cd %USERPROFILE%\Desktop\contribution-margin-system && claude
  3. Tell Claude Code to set up the system. Once you see the ❯ prompt, paste: set up the cm system Claude Code will install Python (if needed), the libraries, copy the skills into place, then walk you through entering your Shopify and Meta credentials and your unit economics (COGS, shipping rates, etc.) interactively. Set aside ~30 minutes for this — most of it is your unit economics, which only you can fill in.
  4. Start asking questions. Once setup completes, just talk to Claude Code in plain English: how much did we make yesterday? show me daily CM for the last 14 days what hours of the day are we losing money?
⚠ Heads up — password prompts

Claude Code may ask for your computer password once or twice during setup (when installing Python or system packages). That's normal — type your password into the Terminal window when you see the prompt. You won't see characters as you type. Press Enter when done.

02What's in the bundle

Three skills + one cron script + the templates Claude Code uses to walk you through unit economics:

cm-tracker

Single-period contribution margin with full waterfall

Pulls Shopify orders + Meta ad spend, calculates real per-order COGS / shipping / processing / returns, outputs a true CM. Default: yesterday. Trigger phrases: "how much did we make today", "what's our CM", "how are we pacing", "ad spend yesterday".

cm-trends

Multi-period comparison + baseline percentiles

Compare any two windows (this week vs last, March vs February, daily for 14 days). Builds a 60–90 day baseline so you can tell if today is normal variance or actually worth investigating. Trigger phrases: "compare this week to last week", "was March better", "is today normal".

hourly-analysis

Hourly profitability — find the money pits

Pulls 90-day CVR by hour from Shopify, cross-references with CM by hour from the snapshot log. Surfaces hours where CVR looks fine but CM is bleeding (the $1K/day insight from the video). Needs at least 7 days of cron data first. Trigger phrases: "what hours are we losing money", "is our evening spend worth it", "CVR vs CM by hour".

The cron script itself runs every 15 minutes silently in the background, writing snapshots to a JSONL log and refreshing the HTML dashboard.

03If something goes wrong

Just describe what you saw to Claude Code. It can read its own scripts and configs, parse error messages, and walk you through fixes interactively — that's the whole point.

The most common issues:

  • Numbers look wrong — almost always your unit-economics.json is off. Re-check processing_rate and returns_rate against your real statements first.
  • Cron isn't running on Mac — macOS may block cron from accessing your project folder. Tell Claude Code "the cron isn't running" and it'll guide you through granting Full Disk Access (or moving the folder).
  • Shopify or Meta returns 401 — your token expired or doesn't have the right scopes. Tell Claude Code which one errored and it'll walk you through regenerating it.
